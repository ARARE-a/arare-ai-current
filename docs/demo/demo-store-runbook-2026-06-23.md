# デモ店舗 本番デモ手順 2026-06-23

## 対象URL

- 本番アプリ: `https://arare-ai-three.vercel.app`
- 店舗ダッシュボード: `https://arare-ai-three.vercel.app/store-v2`
- 予約管理: `https://arare-ai-three.vercel.app/reservations`
- 電話AIログ: `https://arare-ai-three.vercel.app/phone-ai#call-logs`
- 通知ログ: `https://arare-ai-three.vercel.app/notification-logs`
- 提出前チェック: `https://arare-ai-three.vercel.app/platform`
- Voice Relay health: `https://voice-relay-production-dd5f.up.railway.app/health`

## 1. デモ用データ整理

デモ直前はデータを削除せず、画面上の表示条件で整理する。

- 予約一覧は `今後のみ` を使う。
- 通知ログは通常表示で `送信済み` 中心に見る。
- 過去の未送信・失敗ログは通常表示から非表示になる実装が入っている。
- 失敗通知を確認する場合だけ、通知ログの状態を `未送信` または `送信失敗` に切り替える。
- 電話AIログは `未対応 0件` を確認し、必要なら `確認済みにする` で処理する。

削除しない理由: 通知、通話、予約は後で原因追跡に使う証跡のため。

## 2. 店舗に見せる順番

1. ホーム
   - 要対応、予約、空室、出勤を見る。
   - デモ時の理想は `要対応 0`、`AI未対応 電話0 / 通知0`。

2. 店舗ダッシュボード
   - 店舗名、本日の予約、空室、出勤を確認する。
   - 部屋が利用中/空きに切り替わることを説明する。

3. 電話AIログ
   - 通話要約、予約ID、内容を確認する。
   - 要確認ログがあれば、確認済みにできることを見せる。

4. 予約確定
   - 予約タブで `今後のみ` を選ぶ。
   - 仮予約を確定する。
   - 確定時に顧客SMSとセラピストLINE通知が作られる。

5. 通知ログ
   - `送信済み`、`失敗 0件` を確認する。
   - `本文を表示` で顧客SMS本文とセラピストLINE本文を確認する。

## 3. 提出前チェック画面

管理タブまたは提出前チェック画面で確認する。

- 管理状態スコアが `100`。
- 提出判定が `提出可能`。
- ブロックが `0件`。
- ただし、LINE/SMSの実到達は画面スコアだけでは完了扱いにしない。

## 4. 本番URLとログイン情報の扱い

- デモURLは `https://arare-ai-three.vercel.app` に固定する。
- デモ用ログインは店舗説明用のアカウントを使う。
- パスワードやAPIキーはこのリポジトリに保存しない。
- Supabase、Twilio、LINE、Railway、Vercel、Clerk の管理画面は運営者用で、店舗デモでは見せない。

## ユーザー確認済みの範囲

2026-06-22から2026-06-23の本番画面確認で、以下はユーザー操作により確認済み。

- 電話AIの予約受付 1から7。
- 電話AIから仮予約が作成される。
- 仮予約の確定で顧客SMSが届く。
- 仮予約の確定でセラピストLINEが届く。
- 予約変更でセラピストLINE通知が届く。
- 予約キャンセルでセラピストLINE通知が届く。
- セラピストLINE ID登録後、出勤予定更新ができる。

- デモ当日のSMS/LINEが実端末に届くこと。
