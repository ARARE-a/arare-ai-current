# ARARE AI 営業マン理解用資料

作成日: 2026-06-27  
対象: メンズエステ店舗向け AI予約受付MVP  
用途: 営業担当がサービス内容、画面、運用フロー、説明上の注意点を理解するための内部資料

## 1. 目的

ARARE AIは、店舗の予約受付、電話AI、Web予約チャット、予約確定、通知送信、通話ログ確認を一つの運用画面にまとめるAI予約受付MVPです。

営業担当は、単に「AIが予約を取る」と説明するのではなく、以下の流れで説明してください。

1. AIが電話/Web/SNS経由の予約希望を受ける
2. AIが日時、コース、担当、部屋、顧客情報を整理する
3. 必要に応じて仮予約として店舗側へ渡す
4. 店舗側が内容を確認して確定する
5. 確定後、顧客やセラピストへ通知する
6. 通話ログ、通知ログ、予約一覧で運用状況を確認する

重要: 完全自動で全予約を確定するシステムとして売らないこと。現在の見せ方は「AI受付 + 店舗確認 + 通知証跡」が安全です。

## 2. 確認状況

### 確認済み

- スマホ版の店舗ダッシュボードが表示されている
- 店舗ダッシュボードで、本日の予約、要対応、空室、出勤、AI受付ログを確認できる
- 電話AI画面で、AI受付番号、Webhook、Relay URL、通話ログを確認できる
- 電話ログから仮予約作成、リンク予約、確定済み予約の状態を確認できる
- 通知履歴で、送信済み、未送信、失敗の件数と本文を確認できる
- 予約作成/編集画面で、顧客名、電話番号、日時、コース、セラピスト、部屋、状態、受付経路を入力できる
- 予約一覧で、今後の予約、確定予約、担当、部屋、料金を確認できる

### 実装上は確認できるが、営業時に断定を避ける項目

- LINE連携は画面・設計上は用意されているが、導入店舗ごとのLINE公式アカウント設定が必要
- SMSは通知ログ上で送信済み表示があるが、実端末到達は顧客側環境にも依存する
- 電話AIは通話ログ上で実通話受付の証跡があるが、店舗ごとの電話番号/Twilio設定が必要
- AIの回答品質はナレッジ、FAQ、NG回答、店舗情報の登録量に依存する

### 未確認/要確認

- 各導入店舗の実LINE公式アカウントでの送受信
- 各導入店舗のTwilio番号での実電話
- 顧客端末でのSMS到達
- PC版の最終営業資料用スクショ
- 料金プラン、初期費用、月額、従量課金の正式条件

## 3. サービス全体像

ARARE AIは、予約受付を「入口」「判断」「確定」「通知」「監視」に分けて管理します。

| 領域 | 役割 | 店舗への説明 |
|---|---|---|
| 電話AI | 電話で予約希望や空き確認を受ける | 営業時間外や接客中の取りこぼしを減らす |
| Web予約チャット | Web上で予約相談を受ける | LINEやWebからの問い合わせを整理する入口 |
| 店舗ダッシュボード | 今日の予約、空室、出勤、要対応を確認 | 店舗スタッフが毎日見るメイン画面 |
| 予約管理 | 予約作成、編集、取消、確定 | 店舗側が最終判断する場所 |
| 顧客管理 | 顧客別の予約履歴、状態を確認 | リピート対応やNG確認の補助 |
| セラピスト管理 | 出勤、担当、対応可能状況を確認 | 誰が空いているかを予約判断に使う |
| ルーム管理 | Room A/B/Cなどの空き確認 | 部屋の重複予約を避ける |
| 通知/SMS | 顧客、セラピストへの通知 | 確定後の連絡漏れを減らす |
| 通話/会話ログ | AI受付の内容を確認 | AIが何を受けたか後から追える |
| 運用監視 | DB、OpenAI、Twilio、Clerk等の状態確認 | 管理者が障害や設定漏れを確認する |
| 権限管理 | オーナー、マネージャー、スタッフを分ける | 店舗ごとに見せる範囲を分ける |
| KnowledgeBase/FAQ/NG回答 | AIが参照する店舗情報を管理 | 店舗ルールをAI回答に反映する |

## 4. 業務フロー

### 標準フロー

1. 顧客が電話またはWeb Chatで問い合わせる
2. AIが「日時」「コース」「担当希望」「名前」「電話番号」などを聞き取る
3. 空き時間、出勤セラピスト、空き部屋を見て受付可否を整理する
4. 予約候補または仮予約を作成する
5. 店舗ダッシュボードに要対応/予約情報が反映される
6. 店舗スタッフが内容を確認し、必要なら編集して確定する
7. 確定後、通知/SMS/セラピスト通知を送る
8. 通知履歴と通話ログに証跡が残る

### 営業での言い方

「AIが受付の一次対応を行い、店舗側が最後に確認して確定するので、完全放置ではなく、取りこぼしを減らしながら安全に運用できます。」

## 6. 機能一覧

| 機能 | 内容 | 営業での訴求 |
|---|---|---|
| 電話AI受付 | 電話で予約希望を聞き取る | 取りこぼし削減 |
| Web Chat受付 | Web上で予約希望を受ける | 予約入口を増やす |
| 予約作成 | 店舗側で手動作成 | 電話以外の予約も一元管理 |
| 仮予約 | AI受付後に店舗確認へ回す | 誤確定を防ぐ |
| 予約確定 | 店舗側で最終確定 | 現場判断を残せる |
| 予約一覧 | 今後の予約を確認 | 日々の確認が楽 |
| セラピスト割当 | 自動/手動で担当設定 | 出勤状況と連動しやすい |
| 部屋割当 | 自動/手動で部屋設定 | 部屋重複を防ぐ |
| 通知履歴 | 送信済み/未送信/失敗確認 | 連絡漏れ防止 |
| 通話ログ | AI通話内容を確認 | 後追い確認が可能 |
| KnowledgeBase | 店舗情報をAI回答に使う | 店舗ごとの回答品質向上 |
| FAQ | よくある質問を管理 | 問い合わせ対応を標準化 |
| NG回答 | AIが答えない内容を管理 | 不適切回答を抑制 |
| 権限管理 | オーナー/マネージャー/スタッフ | 店舗運用に合わせた管理 |

## 7. 導入時に必要なもの

- 店舗名
- 店舗電話番号
- 住所
- 営業時間
- コース名、時間、料金
- セラピスト名、指名料、出勤情報
- 部屋数、部屋名
- 予約ルール
- キャンセルルール
- NG回答ルール
- 電話AI用の電話番号/Twilio設定
- SMS送信用設定
- LINE連携を使う場合はLINE公式アカウント情報
- 管理者/スタッフのログインメール

## 8. 運用上の注意

- AI受付内容は必ず店舗側で確認する
- 仮予約は確定ではない
- 確定後に通知が送信される流れを説明する
- SMSは送信済みでも端末到達までは別確認
- 電話AIの品質は店舗情報、FAQ、NG回答の整備で上がる
- 実番号や顧客名を営業資料に出す場合は伏せ字加工する
- 性的サービス、違法行為、グレーな内容をAIが許可するような説明は禁止

## 9. 未確認/要確認事項

- 料金プラン
- 初期設定代行の範囲
- 電話AIの月間通話上限
- SMS従量課金の扱い
- LINE公式アカウント接続の標準手順
- PC版営業資料用スクショ
- 店舗ごとの本番電話番号移行手順
