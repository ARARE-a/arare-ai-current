# ARARE AI 日本語PRD全文

## 0. プロダクト名

ARARE AI

メンズエステ向け AI受付・予約自動化OS。

## 1. 目的

ARARE AIは、メンズエステ店舗の受付、予約、顧客管理、セラピスト管理、通知、会話履歴、売上管理を自動化・一元管理するSaaSである。

対象チャネルは以下。

- 電話AI受付
- LINE AI受付
- WebチャットAI受付
- 管理画面
- 店舗スタッフ向けスマホ操作画面

重要: これは単なるAI予約システムではない。店舗ごとの情報をAIが参照できる「ナレッジ管理機能」を必ず含める。

## 2. 最重要原則

AIは想像で回答してはいけない。

AIが回答に使ってよい情報は以下のみ。

- 店舗設定
- コース
- オプション
- セラピスト情報
- FAQ
- ナレッジ
- トークスクリプト
- NG回答ルール

未登録情報については、必ず以下のように回答する。

「確認が必要です」

AIは顧客の明確な同意なしに予約を確定してはいけない。

## 3. 想定ユーザー

### 3.1 Platform管理者

複数店舗の導入状態、外部連携状態、提出判定、障害、環境変数、確認証跡を管理する。

### 3.2 店舗オーナー

店舗設定、権限、予約、通知、電話AI、LINE、売上、運用品質を管理する。

### 3.3 マネージャー

日常運用、予約確定、顧客対応、通知再送、シフト、部屋割当を管理する。

### 3.4 スタッフ

本日の予約、AI受付ログ、通知失敗、未対応キュー、顧客メッセージを確認・処理する。

### 3.5 セラピスト

自分に割り当てられた予約、出勤、来店準備、連絡事項を確認する。

## 4. プロダクトの方向性

PC版は、管理者・オーナー・マネージャー向けの「司令塔」。

スマホ版は、現場スタッフ・セラピスト向けの「今すぐ対応する作業端末」。

参考UIの方向性:

- PC: Square Dashboard、Shopify Admin、Google Calendar、LINE公式アカウント管理画面
- スマホ: LINE公式アカウントアプリ、Instagram DM、Instagramストーリー、Uber Eats店舗アプリ、Square POS

ただし、既存アプリをコピーしてはいけない。操作感の参考として使う。

## 10. AI受付

### 10.1 電話AI

電話AIは以下を行う。

- Twilio Voice webhook
- 着信応答
- 音声会話
- コース案内
- 料金案内
- セラピスト案内
- 空き確認
- 予約受付
- 予約変更相談
- キャンセル相談
- 最終復唱
- 店舗確認へのエスカレーション
- 通話ログ保存
- 通話文字起こし保存
- 会話要約
- AI聞き取り信頼度の保存
- 仮予約内容の保存
- 要確認事項の保存

電話AIは、顧客の明確な同意と安全な承認フローなしに予約確定してはいけない。

MVP段階では、電話AIは「一次受付」「仮予約作成」「店舗通知」「通話ログ保存」までを主目的にする。

電話AIで最初から行わないこと:

- 電話AIによる即時予約確定
- クレームの自動解決
- 値引き交渉
- 複雑なコース提案
- NG客のAI自動判定
- 売上改善提案
- セラピストへの個別相談
- 長時間の雑談対応
- 予約ルール外の特殊対応

電話AIで必ず確認する項目:

- 希望日時
- コース
- 指名有無
- 顧客名
- 折り返し可能な電話番号
- 来店経験
- 注意事項確認

電話は聞き間違いが起きるため、以下は必ず復唱確認する。

- 日時
- コース
- 顧客名
- 電話番号

Twilioで発信者番号を取得できる場合でも、非通知や別番号の可能性があるため、折り返し可能な電話番号を必ず確認する。

## 11. 予約確定フロー

予約確定は必ず以下の順番にする。

1. 希望日時を取得
2. コースを取得
3. 指名有無を取得
4. 顧客名を取得
5. 電話番号を取得
6. 来店経験を取得
7. NGフラグ・ブラックリストを確認
8. 対応可能セラピストを検索
9. セラピストの空きを確認
10. 部屋の空きを確認
11. ダブルブッキングを検証
12. 仮予約を作成
13. AIが予約内容を復唱
14. 顧客が明確に同意
15. 店舗へ確認通知
16. 店舗または承認APIで確定
17. SMSまたはLINEで予約確定通知を送信

AIは仮予約を飛ばしてはいけない。

AIは直接確定予約を作ってはいけない。

予約作成は以下を使う。

- TENTATIVE
- ReservationHold
- approveReservation などの承認フロー

確定には以下が必要。

- 有効なhold
- 期限内
- 却下されていない
- 空き再確認

## 11.1 人間確認フロー

MVPでは、AIは原則として予約を即時確定しない。

基本フローは以下とする。

1. AIが必要情報を取得
2. AIが空き状況を確認
3. AIが仮予約を作成
4. AIが予約内容を復唱
5. 店舗管理者またはマネージャーが内容を確認
6. 店舗側が承認した場合のみ確定
7. 確定通知を送信

理由:

- 予約ミスを防ぐ
- NG客やブラックリスト対応を店舗側で確認する
- 当日キャンセルや特殊希望をAIが勝手に処理しないようにする
- 電話AIの聞き間違いによる事故を防ぐ

店舗設定で自動確定を有効にした場合のみ、条件を満たす予約を自動確定できる。

ただし、自動確定を許可する場合でも以下をすべて満たすこと。

- 店舗の初回設定チェックリストが完了している
- 顧客がNG/ブラックリストではない
- 予約ルール内である
- キャンセル/変更期限ルールに違反していない
- コース、料金、セラピスト、部屋が登録済み
- セラピストが出勤中または出勤予定
- 部屋が空いている
- 顧客が明確に同意している
- NotificationLogが作成できる
- AuditLogが作成できる

上記のどれか1つでも満たさない場合は、自動確定せず店舗確認待ちにする。

## 11.2 AIから人間へのエスカレーション

AIは以下の場合、無理に回答・確定・キャンセル・変更を行わず、店舗へ通知する。

- 不明点がある
- 顧客の入力が曖昧
- クレーム
- 特殊希望
- NGワード
- 予約ルール外の問い合わせ
- 値引き要求
- 個人LINEなど私的連絡先の要求
- ブラックリストまたはNG客の可能性
- 料金やコースが登録情報と一致しない
- 当日キャンセル
- 期限外の予約変更
- セラピストや部屋の空き判定に不確実性がある
- AIの信頼度が低い

エスカレーション時に必要なもの:

- Conversation
- Message
- Escalation または未対応キュー
- 店舗向け通知
- AuditLog
- 理由
- 元メッセージ
- AIが抽出した予約候補

スマホUIでは、未対応キューとして最上部に表示する。

PC UIでは、AI受付ログまたは運用監視画面に表示する。

## 11.3 予約変更・キャンセル期限ルール

店舗ごとに以下を設定できること。

- 何時間前まで予約変更可能か
- 何時間前までキャンセル可能か
- 当日キャンセルをAIが受け付けてよいか
- 当日キャンセルは店舗確認必須にするか
- 無断キャンセル扱いの条件
- キャンセル料や注意文の案内
- 変更回数の制限

AIは期限外の変更・キャンセルを勝手に確定してはいけない。

期限外または当日キャンセルの場合は、原則として店舗確認待ちにする。

## 11.4 AI応答プレビュー / テストモード

店舗管理者が、AI受付開始前に「この質問にAIがどう返すか」を確認できる画面を作る。

必要な機能:

- テスト質問入力
- Web Chat / LINE / 電話AI相当の応答プレビュー
- 参照したナレッジ、FAQ、コース、セラピスト、店舗設定の表示
- 未登録情報の場合に「確認が必要です」と返る確認
- NGワード、値引き要求、個人LINE要求のエスカレーション確認
- 予約候補抽出結果の表示
- 店舗確認待ちになる理由の表示

このテストモードでは実予約、実SMS、実LINE送信を行わない。

## 11.5 手動予約ブロック

店舗側が任意の時間帯を予約不可にできること。

必要なブロック種別:

- 店舗全体の予約不可
- 部屋単位の予約不可
- セラピスト単位の予約不可
- 休憩
- 清掃
- メンテナンス
- 貸切
- 管理者都合

ブロックされた時間帯は、AI予約、LINE予約、Webチャット予約、電話AI予約、管理画面予約のすべてで予約不可にする。

## 11.7 非通知電話の拒否

非通知からの電話は原則としてAI受付対象外にする。

必須仕様:

- caller number が非通知、unknown、anonymous、private、blocked などの場合は予約受付しない
- 非通知には「番号通知のうえおかけ直しください」と案内する
- 非通知からの予約作成、予約変更、キャンセルは受け付けない
- 非通知着信はCallLogまたはAuditLogに残す
- 店舗画面に非通知着信件数を表示できるようにする

店舗設定で非通知拒否を変更できる場合でも、MVPではデフォルトを拒否にする。

必須機能:

- 予約作成
- 予約編集

- シフト外予約防止
- 部屋不足防止
- 変更履歴
- 監査ログ

予約ステータス:

- TENTATIVE
- CONFIRMED
- VISITED
- CANCELLED
- NO_SHOW

ダブルブッキング防止:

- 同じセラピストに重複する予約を入れない。
- 同じ部屋に重複する予約を入れない。
- 予約確定処理は可能な限りDBトランザクションで行う。

シフト外予約:

- セラピストが出勤予定または出勤中でなければ予約不可。

部屋不足:

- 空き部屋がなければ予約不可。

## 23. 必須エラー対策

### 23.1 ダブルブッキング

同じセラピストまたは同じ部屋に重複予約を作らない。

### 23.2 シフト外予約

出勤していないセラピストへの予約を拒否する。

### 23.3 部屋不足

空き部屋がない場合は予約不可。

### 23.4 AIの勝手な予約確定

仮予約、復唱、明確な同意、承認フローなしに確定しない。

### 23.5 幻覚回答

AIは登録済み情報だけで回答する。

### 23.6 存在しないコース案内

Courseに存在しないコースを案内しない。

### 23.7 勝手な値引き

AIは値引き交渉や割引提案をしない。

### 23.8 LINE通知失敗

失敗理由をNotificationLogへ保存する。

### 23.9 通知重複

NotificationLogで重複送信を防ぐ。

### 23.10 タイムゾーン

表示はJST固定。

DB保存時の日時扱いも統一する。

### 23.11 顧客重複

電話番号を店舗単位でユニークにする。

### 23.12 予約変更ミス

変更前後の履歴を保存する。

### 23.13 キャンセル済み売上計上

キャンセル、無断キャンセルは売上に入れない。

### 23.14 電話AIの聞き間違い

日時、コース、名前、電話番号を必ず復唱する。

### 23.15 本人以外の予約変更

電話番号と予約情報を照合する。

### 23.16 OpenAIコスト増加

毎回全履歴を送らない。必要に応じて要約を使う。

### 23.17 権限漏れ

セラピストが全顧客情報を見られないようにする。

### 23.18 人間確認漏れ

AIが店舗確認なしに予約確定しないようにする。

自動確定を使う場合でも、店舗設定で明示的に有効化され、条件を満たす場合のみに限定する。

### 23.19 エスカレーション漏れ

不明点、クレーム、特殊希望、NGワード、ルール外問い合わせは必ず店舗へ通知する。

### 23.20 変更・キャンセル期限違反

店舗が設定した期限を過ぎた変更やキャンセルを、AIが勝手に受理しない。

### 23.21 監査ログ不足

予約作成、変更、キャンセル、承認、通知再送、設定変更について、誰が、いつ、何を行ったかをAuditLogまたは変更履歴に残す。

### 23.22 手動ブロック無視

店舗、部屋、セラピスト単位の手動予約ブロックを、AIと管理画面の両方で必ず尊重する。

### 23.23 初回設定不足

必須設定が不足している店舗では、AI受付を開始できないようにする。

### 23.24 非通知電話

非通知、anonymous、private、blocked、unknownの電話番号からの予約作成、変更、キャンセルを拒否する。
