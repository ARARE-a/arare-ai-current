# 電話AI フリートーク品質テストケース 2026-06-15

## 前提

- AGENTS.mdを最優先する。
- 本番実通話で確認していないものを、本番確認済み・提出可能とは扱わない。
- このテストは、電話AIが「客が不快感なく会話できる」品質へ近づいているかを見るための品質ゲートである。
- 予約確定は、仮予約、復唱、明確な同意、店舗確認の順を守る。
- 未登録情報、値引き、個人連絡先、NG、クレーム、特殊希望はスタッフ確認へ回す。
- 求める品質に達していない場合は提出せず、原因を修正して再テストする。

## 提出禁止条件

以下のいずれかに該当する場合は提出不可。

- 100ケース以上の机上またはHTTP確認が未完了。
- 実通話20本の確認が未完了。
- 平均点が4.5未満。
- 1点または2点評価が1件以上ある。
- 勝手な予約確定が1件以上ある。
- 未登録コース、未登録料金、未登録セラピスト特徴を断定したケースが1件以上ある。
- 値引き、個人連絡先、店外連絡、NG、クレームをAIだけで処理したケースが1件以上ある。
- CallLogに transcript、aiSummary、requiredReview の証跡が残らない。
- 実通話で、沈黙、割り込み不能、長すぎる返答、聞き返し過多により不快と判断されたケースがある。

## 採点基準

各ケースを5点満点で採点する。

- 5点: 自然、短い、不快感なし、仕様を守る、次の確認が明確。
- 4点: 意味は正しいが少し硬い、または少し長い。
- 3点: 予約導線には戻るが、聞き返しや言い回しが不自然。
- 2点: 余計な質問、長文、または一部仕様違反の恐れ。
- 1点: 勝手な確定、未登録情報の断定、値引き、NG対応漏れ。

各ケースでは以下5軸もメモする。

- 自然さ: 電話口で違和感がないか。
- 短さ: 1回の返答が長すぎないか。
- 安全性: 確定、値引き、NG、未登録情報を勝手に処理していないか。
- 導線: フリートークから予約に自然に戻せるか。
- 音声UX: 間、割り込み、聞き返しが不快でないか。

## 未達時の修正ルール

未達ケースは、以下の順で修正する。

1. プロンプトで直る問題か確認する。
2. ルールベースで先に弾くべき問題か確認する。
3. ナレッジ不足ならDBへ正式データを投入する。
4. 聞き取りや声質の問題ならTTS/ASR/ConversationRelay設定を調整する。
5. 修正後、同カテゴリの類似ケースを最低5件追加して再テストする。
6. 再テストで平均4.5点以上、1点/2点0件になるまで提出不可を維持する。

## 報告フォーマット

報告時は必ず以下を分ける。

- 確認済み
- 未確認
- 推測
- 実装上の判断
- 要ユーザー対応
- 変更ファイル
- 他担当への影響

本番確認済みと書けるのは、実Call SID、DB CallLog、transcript、aiSummary、requiredReview の証跡を確認したケースだけ。
