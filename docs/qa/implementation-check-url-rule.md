# 実装後チェックURL提出ルール

実装作業を行った後は、報告時に以下を必ず分けて記載する。

- 確認済み: ローカルで実行した検証、ビルド、デプロイ、ログ確認など。
- 未確認: ユーザー操作や外部サービス反映など、こちらで最後まで確認できていない項目。
- 確認用URL: ユーザーが本番画面で確認するためのURL。

本番URLで実際に確認していない項目は、本番確認済みと書かない。

基本の確認用URL:

- ダッシュボード: https://arare-ai-three.vercel.app/store-v2
- 予約管理: https://arare-ai-three.vercel.app/reservations
- 通知ログ: https://arare-ai-three.vercel.app/notification-logs
