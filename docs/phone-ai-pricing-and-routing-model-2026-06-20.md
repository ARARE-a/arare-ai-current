# 電話AIの料金設計と切替運用モデル 2026-06-20

担当: Codex

## 結論

電話AIは、常時すべての電話をAIで受ける前提にすると通話分数が読みにくく、店舗への説明も難しくなる。

ARARE AIでは、電話AIを必要時だけONにできる前提で料金設計する。

- 営業時間外のみAI
- 接客中/不在/応答なし時のみAI
- 混雑時のみAI
- 手動運用時はAIをOFF

この形なら、電話AIは「電話代が増えるサービス」ではなく、「予約取りこぼしを防ぐ受付補助」として説明できる。

## なぜ受信側にTwilio費用が発生するか

顧客が店舗へ電話した場合でも、AI受付番号または転送先がTwilioを通ると、Twilioアカウント側に受信通話料と音声AI処理費が発生する。

これは顧客の携帯電話料金とは別で、ARARE AI側が電話AI基盤を使うためのインフラ費である。

## 原価目安

2026-06時点のTwilio日本向け公開料金を前提にした概算。

| 項目 | 目安 |
| --- | ---: |
| Twilio着信通話 | $0.0100 / 分 |
| Conversation Relay | $0.0700 / 分 |
| Twilio系合計 | $0.0800 / 分 |
| 円換算目安 | 約13円 / 分 |
| LLM/運用余裕込みの管理目安 | 15〜25円 / 分 |

## 推奨料金案

初期の説明では、Standardを中心にする。

| プラン | 月額 | AI電話受付 | 超過 |
| --- | ---: | ---: | ---: |
| Lite | 19,800円 | 月100分まで込み | 50円 / 分 |
| Standard | 29,800円 | 月300分まで込み | 50円 / 分 |
| Pro | 49,800円 | 月800分まで込み | 50円 / 分 |

推奨の主力説明:

```text
月額29,800円。AI電話受付300分込み。
営業時間外や接客中など、必要な時だけAIに切り替えできます。
通常運用では追加費用を気にせず使え、通話量が多い場合のみ超過分を1分単位で精算します。
```

## 店舗への説明文

```text
電話AIは常時稼働にする必要はありません。
営業時間外、接客中、電話に出られない時だけAIに切り替えできます。
月額料金には一定分数のAI受付枠を含めているため、通常利用では追加費用を気にせず運用できます。
```

## 実装上の判断

- `StorePhoneSetting.routingMode` で「常時AI」「営業時間外のみAI」「不在/応答なし時AI」「手動のみ」を管理する。
- `StoreUsageMeter` で店舗ごとの月次AI通話秒数を管理する。
- Twilio終了callbackで `CallDuration` が取得できる場合、`CallLog.durationSeconds` と `StoreUsageMeter.voiceCallSeconds` に保存する。
- 同一CallSidの二重計上を避けるため、`CallLog.usageMeterRecordedAt` が空の場合だけ月次メーターへ加算する。

## 未確認

- Twilio本番callbackが常に `CallDuration` を送るかは、実CallSid付きで確認が必要。
- Railway側の音声Relayが独自に通話終了処理を持つ場合、このリポジトリ側と同じメーター更新処理を反映する必要がある。
- 実際の請求単価は為替、Twilio課税、番号種別、録音、LLM使用量で変動する。
