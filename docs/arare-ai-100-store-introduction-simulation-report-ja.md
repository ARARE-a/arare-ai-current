# ARARE AI 100店舗導入シミュレーション報告書

作成日: 2026-06-12

対象: 店舗側、ARARE AI運営/設計側

前提:

- 導入規模は100店舗
- 店舗の既存電話番号は変更しない
- MVPでは、既存店舗番号から店舗専用のTwilio番号へ転送する
- 店舗ごとに `aiIngressPhoneNumber` を1つ持つ
- 顧客に見える番号は既存店舗番号のまま
- AIは予約確定を直接行わず、仮予約と店舗承認までを担う
- 為替は概算で 1 USD = 160 JPY
- Twilio料金は2026-06-12時点の公式料金を前提

---

## 1. エグゼクティブサマリー

ARARE AIは、100店舗導入を前提にしても、店舗ごとにAI受付用番号を持つ構成が現実的である。

最も推奨する導入方式は以下。

```text
顧客
  ↓ 既存店舗番号に電話
店舗の既存電話番号
  ↓ 電話会社/PBX/スマホ転送
店舗専用Twilio番号
  ↓
ARARE AI 電話AI
  ↓
仮予約作成
  ↓
店舗スタッフ承認
  ↓
予約確定通知
```

この方式なら、店舗は公開番号を変えずに導入できる。

100店舗時点のARARE AI側の番号固定費は、National番号を使う場合で月約76,000円。

ただし、本番運用で主な変動費になるのは番号代ではなく、通話分数に応じたTwilio通話料とConversation Relay費用である。

目安として、100店舗全体で1店舗あたり月500分のAI通話がある場合、Twilio番号代 + 通話 + Conversation Relayで月約716,000円程度になる。

## 2. 店舗側から見た導入シミュレーション

### 2.1 店舗側の導入メリット

店舗側の主なメリット:

- 店舗の電話番号を変えなくてよい
- 営業中、施術中、深夜帯でも一次受付できる
- 電話、LINE、Webチャットの問い合わせを一元管理できる
- 顧客対応漏れを減らせる
- 予約内容がログ化される
- スタッフは「承認」中心の操作になる
- セラピストのシフト登録をLINEから受けられる
- 店舗ごとのコース、料金、NGルール、注意事項をAIが参照できる

### 2.2 店舗側の導入手順

店舗側で必要な作業:

1. 既存電話番号の転送設定を確認
2. ARARE AIから発行されたAI受付番号へ転送設定
3. 店舗情報を登録
4. コース、料金、部屋、セラピスト、シフトを登録
5. FAQ、注意事項、NG対応ルールを登録
6. LINE公式アカウントを接続
7. 店舗スタッフ、マネージャー、オーナーを登録
8. テスト着信
9. テストLINE問い合わせ
10. 仮予約から承認までのテスト
11. 本番開始

### 2.3 店舗側の費用項目

店舗側で発生する可能性がある費用:

| 項目 | 説明 | 備考 |
|---|---|---|
| ARARE AI月額利用料 | ARARE AIに支払う月額 | 料金設計次第 |
| 既存電話会社の転送サービス料 | 店舗番号からAI番号へ転送するための費用 | 電話会社/PBX契約に依存 |
| 転送通話料 | 店舗番号からAI番号へ転送される通話料 | 店舗側の電話会社に発生する可能性 |
| LINE公式アカウント費用 | LINE側のプラン費用 | 利用量・プラン次第 |
| 初期設定作業 | 店舗情報、コース、セラピスト等の登録 | 代行するなら初期費用化可能 |

店舗への説明文:

```text
店舗番号は変えずに導入できます。
ただし、既存電話会社の転送設定と、電話会社側の転送料金が発生する場合があります。
```

### 2.4 店舗側の運用イメージ

電話予約の場合:

1. 顧客が既存店舗番号に電話
2. 電話がARARE AIへ転送
3. AIが要件を聞き取る
4. AIが店舗ナレッジを参照
5. AIが希望日時、コース、指名、名前、電話番号を確認
6. AIが仮予約を作成
7. 店舗スタッフに通知
8. スタッフがスマホまたはPCで承認
9. 顧客に確定通知

LINE予約の場合:

1. 顧客がLINEで問い合わせ
2. AIが登録済みナレッジだけで回答
3. 予約候補を抽出
4. 仮予約を作成
5. スタッフ承認後に確定

セラピストのシフト登録:

1. セラピストがLINEで出勤表を送信
2. AIが解析
3. `LineShiftParseJob` として保存
4. スタッフまたはセラピストが確認
5. Shiftへ反映

### 2.5 店舗側のリスクと対策

| リスク | 対策 |
|---|---|
| 電話転送設定が難しい | 導入チェックリストとテスト着信画面を用意 |
| 転送料金が店舗側に発生 | 事前に店舗の電話会社へ確認してもらう |
| AIの聞き間違い | 日時、コース、名前、電話番号を必ず復唱 |
| AIが勝手に予約確定 | AIは仮予約まで。確定はスタッフ承認制 |
| 店舗確認が遅れる | 5分再通知、10分強通知、15分でhold期限切れ |
| セラピストの自由文シフトが誤解析 | MVPではテンプレ入力優先。自動反映せず確認後反映 |
| NG客対応 | criticalのみ自動ブロック。high以上はスタッフ承認必須 |

## 3. ARARE AI運営/設計側から見た導入シミュレーション

### 3.1 基本構成

100店舗導入時の標準構成:

- 1店舗 = 1つの `Store`
- 1店舗 = 1つの `aiIngressPhoneNumber`
- 1店舗 = 1つ以上のLINE連携設定
- 1店舗 = 独立したナレッジ、FAQ、NGルール
- 1店舗 = 独立した予約、顧客、セラピスト、シフト、部屋

MVPでは、店舗識別は `aiIngressPhoneNumber` を優先する。

理由:

- 転送元情報が保持されない電話会社/PBXがある
- 複数店舗が同じAI受付番号へ転送すると、店舗判定が不安定になる
- 100店舗では誤ルーティングが重大事故になる

### 3.2 ARARE AI側の主な原価

ARARE AI側で発生する費用:

| 項目 | 説明 |
|---|---|
| Twilio番号月額 | 店舗ごとのAI受付番号 |
| Twilio着信通話料 | 電話AIに入ってくる通話 |
| Conversation Relay | リアルタイム音声AI処理 |
| OpenAI/LLM | 会話生成、要約、抽出 |
| TTS/STT追加費用 | 外部音声プロバイダを使う場合 |
| 録音・ストレージ | 通話録音、文字起こし、ログ |
| DB/サーバー | PostgreSQL、API、ジョブ、管理画面 |
| 監視/アラート | 外部連携失敗、通知失敗、未承認hold |
| サポート人件費 | 導入支援、設定代行、障害対応 |

### 3.3 Twilio番号固定費

100店舗に導入し、店舗ごとにAI受付用Twilio番号を1つ持つ場合。

| 番号種別 | Twilio月額 | 100店舗分 | 円換算 |
|---|---:|---:|---:|
| National numbers | $4.75 / 月 | $475 / 月 | 約76,000円 / 月 |
| Local 0ABJ numbers | $20.00 / 月 | $2,000 / 月 | 約320,000円 / 月 |
| Toll-free numbers | $25.00 / 月 | $2,500 / 月 | 約400,000円 / 月 |

推奨:

- MVPはNational numbers
- 顧客に直接見せる番号ではないため、番号種別の見た目よりも費用効率を優先
- 0ABJやToll-freeは本格導入、ブランド要件、直通番号として使う場合に検討

### 3.4 AI電話1分あたりの概算原価

National番号またはLocal番号で受ける場合:

| 項目 | 単価 |
|---|---:|
| Twilio着信通話 | $0.0100 / 分 |
| Conversation Relay | $0.0700 / 分 |
| 合計 | $0.0800 / 分 |
| 円換算 | 約12.8円 / 分 |

含めていないもの:

- OpenAI/LLM費用
- 外部TTS費用
- 録音
- ストレージ
- 人間スタッフへ電話転送する場合の発信通話料
- 店舗側の電話会社に発生する転送料金
- 税、為替手数料

前提:

### 3.6 月額課金別の粗利シミュレーション

前提:

- 100店舗
- National番号
- Twilio番号 + 着信 + Conversation Relayのみを原価計上
- LLM、録音、外部TTS、サーバー、人件費は除外

| 月額単価/店舗 | 月商 | 100分/店 原価後 | 500分/店 原価後 | 1,000分/店 原価後 | 2,000分/店 原価後 |
|---:|---:|---:|---:|---:|---:|
| 29,800円 | 2,980,000円 | 約2,776,000円 | 約2,264,000円 | 約1,624,000円 | 約344,000円 |
| 49,800円 | 4,980,000円 | 約4,776,000円 | 約4,264,000円 | 約3,624,000円 | 約2,344,000円 |
| 79,800円 | 7,980,000円 | 約7,776,000円 | 約7,264,000円 | 約6,624,000円 | 約5,344,000円 |

示唆:

- 月額29,800円は、低〜標準利用なら成立しやすい
- 月額29,800円で1店舗2,000分を含めると薄い
- 月額49,800円以上なら、AI通話量が多い店舗にも耐えやすい
- 通話分数の上限、または超過課金を設けた方が安全

推奨する料金設計:

- 基本月額 + 含まれるAI通話分数
- 超過分は従量課金
- 高通話店舗には上位プラン

例:

| プラン | 月額 | 含むAI通話 | 超過 |
|---|---:|---:|---:|
| Lite | 29,800円 | 300分 | 25〜40円/分 |
| Standard | 49,800円 | 800分 | 25〜40円/分 |
| Pro | 79,800円 | 2,000分 | 25〜40円/分 |

これは仮案。正式価格は、LLM費用、サポート費用、営業コストを入れて再計算する。

## 4. 100店舗導入時のオペレーション設計

### 4.1 導入フェーズ

推奨フェーズ:

| フェーズ | 店舗数 | 目的 |
|---|---:|---|
| Phase 0 | 1〜3店舗 | デモ、内部検証、電話転送テスト |
| Phase 1 | 5〜10店舗 | パイロット、予約事故防止、運用確認 |
| Phase 2 | 30店舗 | 監視、通知、サポート手順の安定化 |
| Phase 3 | 100店舗 | 本格展開、Platform Admin運用 |

いきなり100店舗本番化しない。

### 4.2 Platform Adminで必要な監視

Platform Admin画面に必須の監視項目:

- 店舗別導入ステータス
- 店舗別電話転送ステータス
- 店舗別AI受付ステータス
- 店舗別LINE連携ステータス
- 店舗別SMS callbackステータス
- 店舗別未承認ReservationHold数
- 店舗別未対応Escalation数
- 店舗別通知失敗数
- 店舗別通話件数
- 店舗別AI応答失敗数
- 外部Webhook最終受信時刻

### 4.3 必須ジョブ

100店舗では、以下をジョブ化できる設計にする。

- ReservationHold期限切れ処理
- スタッフ承認SLA再通知
- 通知送信
- 通知再送
- 前日リマインド
- CallLog要約
- LINEシフト解析
- 外部連携ヘルスチェック

MVPではDBベースの簡易ジョブでよい。

### 4.4 サポート体制

100店舗時点で想定される問い合わせ:

- 電話転送できない
- LINE webhookが動かない
- AIが店舗情報を知らない
- セラピストのシフトが反映されない
- 通知が届かない
- 予約承認を忘れた
- 顧客が予約確定だと誤解した

必要な運用:

- 店舗別導入チェックリスト
- 外部連携チェックリスト
- テスト着信手順
- LINEテスト送信手順
- 予約承認テスト手順
- 障害時の一次切り分け手順

## 5. 導入時の説明資料に入れるべき文言

### 5.1 店舗向け説明

```text
ARARE AIは、既存の店舗電話番号を変えずに導入できます。
店舗の電話番号にかかってきた電話を、店舗専用のAI受付番号へ転送することで、AIが一次受付します。
顧客に見える電話番号は今まで通りです。
AIは勝手に予約確定せず、仮予約を作成し、店舗スタッフの承認後に確定します。
```

### 5.2 注意文

```text
電話転送の設定は、店舗が契約している電話会社、固定電話、IP電話、PBX、スマホ転送機能に依存します。
転送サービスの月額費用や転送通話料が店舗側に発生する場合があります。
```

### 5.3 AI受付の安全説明

```text
AIは登録済みの店舗情報、コース、料金、セラピスト、シフト、部屋、FAQ、注意事項、NGルールだけを参照します。
未登録情報については「確認が必要です」と回答し、店舗スタッフへ確認依頼します。
```

## 6. 推奨結論

100店舗導入前提でも、店舗ごとにAI受付用Twilio番号を持つ構成は現実的。

MVPの推奨構成:

- 店舗の公開番号は変えない
- 店舗ごとにNational番号を1つ発行
- 既存店舗番号から店舗専用Twilio番号へ転送
- 店舗判定は `aiIngressPhoneNumber` で行う
- AIは仮予約まで
- 確定は店舗承認
- 100店舗分の導入状態をPlatform Adminで監視

費用面:

- 番号固定費は100店舗で約76,000円/月から始められる
- 変動費は通話分数に比例する
- 1店舗500分/月なら、Twilio系原価は1店舗あたり約7,160円/月
- 月額29,800円でも標準利用なら粗利は出る可能性がある
- 高通話店舗向けに上位プランまたは超過課金は必要

次に決めるべきこと:

1. ARARE AIの月額プラン
2. 含まれるAI通話分数
3. 超過通話単価
4. 初期設定費用を取るか
5. Twilio/LLM費用をARARE側が吸収するか、従量で請求するか
6. パイロット店舗数
7. 店舗側の電話転送設定を誰が支援するか
